# Hockey Runner MVP

Первичный MVP для тестирования идеи:

- телефон-трекер с камерой;
- ручная калибровка 4 углов светлого поля;
- распознавание темной круглой шайбы диаметром около 7.5 см;
- фильтр от клюшки по форме, размеру, aspect-ratio и fill-ratio;
- экран игры на втором устройстве;
- локальная связка устройств через relay WebSocket на MacBook;
- старт игры через 15 секунд после кнопки «Начать игру».

## Что важно

Это тестовый пакет. Он специально сделан так, чтобы быстро проверить механику на iPhone 15 + MacBook/Xcode без написания нативного OpenCV-модуля. Камера и обработка кадра работают внутри WebView/Canvas. Для production-качества следующий шаг — заменить этот слой на нативный frame processor: iOS Swift/AVFoundation/OpenCV и Android Kotlin/CameraX/OpenCV или VisionCamera native plugin.

## Быстрый запуск

```bash
npm install
npx expo install --fix
npm run relay
```

Во втором терминале:

```bash
npm run prebuild:ios
open ios/*.xcworkspace
```

В Xcode выберите физический iPhone, Team в Signing & Capabilities и нажмите Run.

## Relay URL

После `npm run relay` терминал покажет адреса вида:

```text
ws://192.168.1.50:8787
```

Именно этот адрес нужно вписать в приложении на iPhone/iPad. `localhost` на физическом iPhone использовать нельзя, потому что это сам iPhone, а не MacBook.

## Первый сценарий теста

1. MacBook, iPhone и iPad/симулятор должны быть в одной Wi‑Fi сети.
2. На MacBook запустите `npm run relay`.
3. На iPhone откройте приложение, укажите Relay URL `ws://IP_MACBOOK:8787`, выберите «Открыть телефон‑трекер».
4. Нажмите «Включить камеру».
5. Наведите телефон сверху на светлое поле.
6. Нажмите 4 угла поля: верх‑лево, верх‑право, низ‑право, низ‑лево.
7. Положите темную шайбу на поле. Вокруг нее должен появиться зеленый круг.
8. На втором устройстве откройте это же приложение, тот же Relay URL и Room, выберите «Открыть экран игры».
9. Нажмите «Начать игру». Старт будет через 15 секунд.
10. Двигайте шайбу клюшкой: влево/вправо — уклонение, вперед — прыжок, назад — приседание.

## Что делать, если `Connected clients: 2`, но игрок не двигается

1. На экране игры проверьте строку `Last command`:
   - если там меняются `left/right/jump/duck`, а персонаж не реагирует — нажмите `Начать игру` и дождитесь полного таймера `15` секунд;
   - если там `—`, значит команды из трекера не доходят.
2. Убедитесь, что **Room** одинаковый на телефоне и на Mac/iPad (чувствительно к регистру и пробелам).
3. На трекере убедитесь, что статус `camera: on`, `ws: connected`, `calib: 4/4`.
4. Убедитесь, что шайба реально детектится (зеленый круг на шайбе стабилен, без постоянных скачков).
5. Нажмите `Переподключить relay` сначала на трекере, потом на экране игры.
6. Если у вас исторически использовались команды `up/down/crouch`, текущая версия игры теперь автоматически маппит их в `jump/duck`.

## Подробный пакет запуска и проверки (пошагово)

1. **Подготовка сети**
   - Подключите телефон и MacBook к одной сети Wi‑Fi.
   - Отключите VPN/прокси на время теста.
2. **Запуск relay**
   - В корне проекта: `npm run relay`.
   - Зафиксируйте адрес вида `ws://192.168.x.x:8787`.
3. **Сборка приложения**
   - `npm install`
   - `npx expo install --fix`
   - `npm run prebuild:ios`
   - `open ios/*.xcworkspace`
   - Запустите на физическом iPhone из Xcode.
4. **Настройка трекера (телефон)**
   - Введите Relay URL и Room.
   - Откройте `Телефон‑трекер`.
   - Нажмите `Включить камеру`.
   - Поставьте телефон на штатив, вид сверху, поле целиком в кадре.
   - Сделайте калибровку 4 углов.
5. **Настройка игры (Mac/iPad)**
   - Введите тот же Relay URL и тот же Room.
   - Откройте `Экран игры`.
   - Нажмите `Начать игру`.
   - Дождитесь окончания отсчета **15 секунд**.
6. **Функциональная проверка управления**
   - Шайба влево → `left`.
   - Шайба вправо → `right`.
   - Шайба вперед (от игрока) → `jump`.
   - Шайба назад (к игроку) → `duck`.
7. **Мини-чек качества CV**
   - Негативный тест: убрать шайбу — команда должна затухать в `neutral/lost`.
   - Шумовой тест: быстрые мелкие дрожания клюшкой не должны давать постоянные ложные команды.
   - Пороговый тест: резкий явный сдвиг шайбы должен стабильно давать одну правильную команду.
