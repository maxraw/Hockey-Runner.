# Hockey Floor Game MVP Starter

Это стартовый пакет для MVP игры, где телефон считывает движение тёмной хоккейной шайбы на светлом коврике и переводит его в управление мини-игрой.

## Что уже заложено

- кроссплатформенная база на Flutter;
- живая камера на iOS и Android;
- CV-обработка через `opencv_dart`;
- калибровка светлого игрового поля;
- поиск тёмной шайбы внутри поля;
- проверка относительного размера поля и шайбы;
- кнопка `Начать игру` после стабильной калибровки;
- отсчёт 15 секунд;
- мини-игра в хоккейном стиле на Flame;
- управление:
  - влево/вправо — уклонение;
  - движение шайбы вперёд — прыжок;
  - движение шайбы назад — приседание.

## Что это пока НЕ делает идеально

Это именно стартовый MVP. Он пригоден для первого запуска и тестирования, но ещё не является финальной production-версией.

Ограничения текущего варианта:

1. Нормализация координат пока идёт по bounding box поля, а не по полноценной перспективной гомографии.
2. Алгоритм рассчитан на хороший контраст:
   - поле заметно светлее пола;
   - шайба заметно темнее поля;
   - ровный свет без жёстких бликов.
3. Сейчас нет записи телеметрии, профиля игрока, меню сложности, сохранения рекордов и финальной стилизации UI.
4. Нет спрайтов персонажа — хоккеист рисуется примитивами, чтобы быстрее запустить механику.

## Рекомендуемая архитектура

- Flutter — оболочка приложения, экраны, логика состояния.
- camera — поток кадров с камеры.
- opencv_dart — CV-обработка кадра.
- Flame — мини-игра.

Поток данных:

1. Камера отдаёт кадры.
2. Кадр конвертируется в RGBA.
3. OpenCV ищет светлое поле.
4. OpenCV ищет тёмную шайбу внутри поля.
5. Координата шайбы нормализуется в диапазон 0..1.
6. По смещению шайбы вычисляются команды.
7. Команды отправляются в игровую сцену.

## Пошаговый запуск с нуля

### Шаг 1. Установи окружение

Нужно:

- Flutter 3.38+;
- Dart 3.10+;
- Android Studio;
- Xcode для iOS;
- CocoaPods на macOS;
- физический телефон Android или iPhone.

Важно: первый build с OpenCV может идти заметно дольше обычного, потому что нативная часть собирается отдельно.

### Шаг 2. Создай новый Flutter-проект

```bash
flutter create hockey_floor_game
cd hockey_floor_game
```

### Шаг 3. Замени файлы проекта файлами из этого пакета

Скопируй из starter-пакета:

- `pubspec.yaml`
- всю папку `lib/`
- `android/app/src/main/AndroidManifest.xml`
- значения из `android/build_gradle_app.kts.snippet`
- строку для `Info.plist` из `ios/Runner/Info.plist.additions.txt`

### Шаг 4. Проверь Android build.gradle.kts

Открой `android/app/build.gradle.kts` и убедись, что:

- `minSdk = 24`
- `namespace` и `applicationId` выставлены корректно

Если у тебя другое имя пакета — просто подставь своё.

### Шаг 5. Добавь доступ к камере на iOS

Открой `ios/Runner/Info.plist` и вставь:

```xml
<key>NSCameraUsageDescription</key>
<string>Камера нужна для распознавания игрового поля и движения хоккейной шайбы.</string>
```

### Шаг 6. Подтяни зависимости

```bash
flutter pub get
```

### Шаг 7. Очисти проект перед первым нативным билдом

```bash
flutter clean
flutter pub get
```

### Шаг 8. Запусти на Android

```bash
flutter run
```

### Шаг 9. Запусти на iOS

На macOS:

```bash
cd ios
pod install
cd ..
flutter run
```

Если build не стартует сразу, открой `ios/Runner.xcworkspace` в Xcode и сделай первый запуск оттуда на реальное устройство.

## Как тестировать физически

### Позиционирование телефона

- Телефон ставится на напольный штатив.
- Камера должна видеть всё поле целиком.
- Желательно, чтобы поле занимало 40–70% высоты кадра.
- Не ставь телефон слишком низко, иначе движения шайбы будут слишком резкими.

### Требования к полю

- светлое;
- матовое;
- без крупных рисунков;
- размер от 100×50 см до 233×100 см.

### Требования к шайбе

- тёмная;
- диаметр около 7.5 см;
- без блестящих отражающих поверхностей.

### Освещение

Лучше всего:

- ровный верхний свет;
- без бликов;
- без резких теней от клюшки и рук.

## Как работает CV в текущем MVP

### 1. Поиск поля

Алгоритм:

- переводит кадр в grayscale;
- размывает изображение;
- бинаризует светлые области;
- ищет внешние контуры;
- выбирает крупнейший прямоугольноподобный контур как поле.

### 2. Поиск шайбы

Алгоритм:

- на том же кадре строит инверсную бинарную маску;
- ищет тёмные контуры;
- оставляет кандидатов только внутри поля;
- отсеивает слишком маленькие и слишком большие объекты;
- выбирает наиболее круглый объект как шайбу.

### 3. Проверка калибровки

Кнопка `Начать игру` активируется только если несколько кадров подряд:

- найдено поле;
- найдена шайба;
- относительный размер шайбы к полю выглядит правдоподобно.

### 4. Команды управления

- смещение по `x` управляет позицией игрока по горизонтали;
- быстрый отрицательный сдвиг по `y` трактуется как прыжок;
- быстрый положительный сдвиг по `y` трактуется как приседание.

## Где менять пороги

Главный файл для тюнинга:

`lib/tracking/cv_tracker_service.dart`

Смотри эти места:

- минимальная площадь поля;
- допустимый aspect ratio поля;
- минимальная/максимальная площадь шайбы;
- минимальная круглотность шайбы;
- диапазон отношения размера поля к шайбе;
- пороги jump/crouch;
- сглаживание координат.

## Что улучшать следующим этапом

### Приоритет 1

1. Добавить перспективное преобразование поля в канонический прямоугольник.
2. Ввести Kalman filter или EMA+velocity smoothing для шайбы.
3. Добавить дебаг-режим с масками `field mask / puck mask`.
4. Разделить игровые режимы: детский / стандарт / быстрый.

### Приоритет 2

1. Заменить примитивного хоккеиста на спрайты.
2. Добавить реальные анимации прыжка и приседания.
3. Добавить меню, рекорды, звук, вибрацию.
4. Сохранять параметры калибровки на устройство.

### Приоритет 3

1. Перейти от простого contour-based трекинга к гибридной схеме:
   - геометрия поля через контуры;
   - шайба через контур + motion consistency;
   - потеря шайбы через re-acquire логику.
2. Добавить запись игровой сессии.
3. Добавить облачную аналитику только после локальной стабилизации MVP.

## Частые проблемы

### 1. Кнопка `Начать игру` не активируется

Причины:

- поле слишком тёмное;
- шайба слишком светлая;
- сильные блики;
- камера видит поле под слишком острым углом;
- шайба слишком маленькая в кадре.

### 2. Прыжок срабатывает слишком часто

Увеличь порог в `_extractCommand()` внутри `cv_tracker_service.dart`.

### 3. Игрок двигается рывками

Усиль сглаживание в `_smooth()`.

### 4. Поле находится неверно

Сузь допустимый диапазон `aspectRatio` и подними минимальную площадь поля.

## Что я бы делал следующим сообщением

После первого запуска я бы перешёл к следующему шагу:

1. включить дебаг-экраны масок;
2. добавить перспективное выпрямление поля;
3. сделать настройку порогов через hidden debug panel;
4. вынести CV в отдельный isolate / отдельный processing pipeline.
